=== MVR API (HTTP REST) ===

BASE: http://{ip}:3333/api/ (Enable: Settings > Advanced > Connections > Remote Access)

AUTH: HTTP Basic with MD5-encrypted password
SESSION: POST /api/begin → X-session header (5min timeout)
VISUAL FEEDBACK: Patient LED slow flash, Camera LED during processing, Video LED during file ops

CORE ENDPOINTS:
Device: GET /api/device (info, no auth), GET /api/cameras, GET /api/storages
Study: PUT /api/study/start (patient metadata), GET /api/study/status, PUT /api/study/snapshot
Recording: PUT /api/study/record/start|stop|pause|resume
Streaming: GET /api/stream/authorize (PIN), GET /api/stream/video?pin= (fMP4), GET /api/stream/audio?pin= (AAC/ADTS)
Files: GET|POST|PUT|DELETE /api/files/<type>/<path>
Notifications: GET /api/notify (chunked JSON: storage/camera/study/recording/PACS/network events)
PACS: PUT /api/pacs/test, POST /api/pacs (schedule upload), GET /api/pacs/errors|scheduled
Worklist: GET /api/worklist (patient lists), GET /api/worklist/notify
System: PUT /api/system/reboot|format|reset_settings, POST /api/system/firmware
Settings: GET|PUT|POST /api/settings
Overlays: POST|PUT|DELETE /api/study/overlays (custom graphics with 9-point align, opacity, rotation)

RS232 SERIAL: <method> <path> [id] [data>\n (auto session per connection)

ERROR CODES: 2xx=Success, 400=Bad params, 401=Unauthorized, ≥300=Failure (plain text)

MUTUAL EXCLUSIVITY: Direct touch vs remote control are mutually exclusive
